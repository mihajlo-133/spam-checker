API Reference

HTTP API for scoring email copy against {{ total }} spam-trigger patterns. Deterministic, stateless, no authentication. Optimized for use by scripts, CI pipelines, and AI agents.

For AI agents: A machine-readable version of these docs is available at /llms.txt (markdown) and /openapi.json (OpenAPI 3.1). Fetch either at session start if your agent generates clients or uses this API dynamically.

Send a POST to /api/check with the email text:

  -H "Content-Type: application/json" \
  -d '{"text": "Hi Sarah, noticed your team is hiring SDRs — quick question."}'

POST /api/check

Check email text and return a scored result.

Request body (JSON)

FieldTypeRequiredDescription
textstringYesEmail subject + body as a single string. Variable placeholders like {{ '{{firstName}}' }} are ignored by the scorer.

Response (200 OK)

FieldTypeDescription
scoreintegerRaw numeric score. Higher = spammier. No upper bound.
ratingenumOne of "Great", "Okay", "Poor". Derived from score.
total_hitsintegerTotal unique (keyword, category) pairs matched.
word_countintegerWord count of the input text (whitespace split).
categoriesobjectKeys are category names. Values are arrays of matched keyword strings.

Categories

Every spam pattern belongs to exactly one of five categories. The category a pattern falls into affects the final score (see Scoring).

CategoryWhat it catchesExample keywords
urgencyPressure and false-scarcity languageAct now, Hurry, Limited time
shadyScam, adult, and high-risk contentFree, Risk-free, Miracle
overpromiseHyperbolic claimsGuaranteed, Best price, Incredible
moneyFinancial-incentive language$$$, Earn, Cash, Income
unnaturalAwkward or bot-written phrasingALL CAPS words, mass-mail artifacts

Scoring algorithm

The score is computed deterministically from the matched categories:

pseudocode 
score = total_hits

if "money" in matched_categories or "shady" in matched_categories:
    score += 20

if "urgency" in matched_categories or "overpromise" in matched_categories:
    score += 10

rating = "Poor"  if score > 20
       = "Okay"  if score > 5
       = "Great" otherwise
Implications:
  • Any single hit in money or shady adds 20 points — pushes to at least "Okay", often "Poor".
  • Any single hit in urgency or overpromise adds 10 points.
  • Hits in unnatural count only as +1 each; they never trigger a category bonus.
  • The same keyword occurring multiple times counts as 1 hit (deduplicated by keyword + category).

Guide for AI agents

When your agent receives a response, apply this decision logic:

RatingRecommended action
"Great"Copy is clean. Ship it.
"Okay"Review flagged words. Rewrite 1–2 specific words. Often acceptable for B2B cold email.
"Poor"High risk. Rewrite required. Inspect categories to see which buckets contributed.

Calibration notes

The engine is aggressive for B2B outbound. Words like "new", "please", "open", "problem" can trigger unnatural.
  • A "Poor" rating driven entirely by unnatural matches — with no money, shady, urgency, or overpromise hits — is usually a false positive for B2B.
  • A "Poor" rating with any money or shady hit is almost always a true positive. Rewrite that passage.
  • Treat this tool as a lexical filter, not a deliverability predictor. It does not check DNS, authentication, reputation, or inbox placement.

When to use this API

  • Check email copy for spam triggers before sending a cold-email campaign.
  • Compare two copy variants and pick the one with the lower score.
  • Filter LLM-generated email drafts for deliverability risk.
  • Audit a library of templates for accumulated spam words.

When NOT to use

  • General deliverability scoring (needs DNS + auth + reputation checks).
  • Inbox placement prediction (needs a seedlist test).
  • Content moderation (tuned for marketing spam, not abuse detection).

Client examples

Python (stdlib only)

python 
import json
import urllib.request

def check_spam(text: str) -> dict:
    req = urllib.request.Request(
        "{{ base_url }}/api/check",
        data=json.dumps({"text": text}).encode(),
        headers={"Content-Type": "application/json"},
        method="POST",
    )
    with urllib.request.urlopen(req, timeout=10) as resp:
        return json.loads(resp.read())

result = check_spam("Act now for a free offer!")
print(result["rating"])  # "Poor"

JavaScript (fetch)

javascript 
const resp = await fetch("{{ base_url }}/api/check", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ text: "Act now for a free offer!" }),
});
const data = await resp.json();
console.log(data.rating); // "Poor"

Batch scoring (parallel requests)

python 
import concurrent.futures

variants = ["variant 1 text", "variant 2 text", "variant 3 text"]
with concurrent.futures.ThreadPoolExecutor(max_workers=5) as ex:
    results = list(ex.map(check_spam, variants))
best = min(results, key=lambda r: r["score"])
print(best)

Operational notes

  • Hosting: Render free tier, Frankfurt region.
  • Cold start: ~30s after 15 min idle. An UptimeRobot HEAD ping every 5 min keeps the service warm during business hours.
  • Warm latency: 250–400ms typical.
  • Persistence: The API does not log or store submitted text.
  • Authentication: None. Public endpoint. Do not submit sensitive PII.
  • Source: github.com/mihajlo-133/spam-checker
  • Engine: 765 regex patterns ported from Mailmeteor (version 2025-04-21). 15/15 rating match against the Mailmeteor web UI.
